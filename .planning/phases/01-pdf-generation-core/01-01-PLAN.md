<task type="auto">
  <name>Task 3: Create font registration module with base64 encoding</name>
  <files>libs/fonts.ts</files>
  <action>
    1. Create libs/fonts.ts file with font registration for @react-pdf/renderer
    2. Use Node.js fs to read the Roboto-Regular.ttf file and convert to base64
    3. Register font with Font.register() at module level (not inside component)
    4. Export the font family name constant for use in PDF stylesheets
    5. Follow this structure:
       ```typescript
       import { Font } from '@react-pdf/renderer';
       import { readFileSync } from 'fs';
       import { join } from 'path';

       // Read font file and convert to base64
       const fontPath = join(process.cwd(), 'public', 'fonts', 'Roboto-Regular.ttf');
       const fontBuffer = readFileSync(fontPath);
       const fontBase64 = fontBuffer.toString('base64');

       // Register font for Vietnamese Unicode support
       Font.register({
         family: 'Roboto',
         src: `data:font/truetype;charset=utf-8;base64,${fontBase64}`,
         // Font includes full Vietnamese character set (Latin Extended-A/B)
       });

       export const ROBOTO_FONT_FAMILY = 'Roboto';
       ```
    6. Add type-safe exports for TypeScript
  </action>
  <verify>
    grep -q "Font.register" libs/fonts.ts && grep -q "Roboto" libs/fonts.ts && grep -q "base64" libs/fonts.ts
  </verify>
  <done>
    - Font.register() called at module level
    - Font loaded from public/fonts/Roboto-Regular.ttf
    - Font converted to base64 data URI
    - Font family exported as ROBOTO_FONT_FAMILY constant
    - No runtime errors when importing the module
  </done>
</task>
